**Автор:** Юлиана Я.

**Цель:** Демонстрация навыков анализа, проектирования API, работы с инцидентами и коммуникации для позиции Integration Specialist.

## Содержание проекта

1.  [Проектирование API отельного продукта](/api-design/) — Анализ и минимальная спецификация в OpenAPI 3.0.
2.  [План коммуникации с недовольным партнером](/communication/escalation_plan.md) — Процедура эскалации и обработки запросов.
3.  [Разбор инцидентов (Case Studies)](/case-studies/) — Анализ трёх реальных кейсов из логов и ответы партнёрам.
4.  [Анализ нагрузки и лимитов API](/api-analysis/rate_limit_analysis.md) — Расчёт и рекомендации по ограничениям.

## Как использовать этот репозиторий

*   **Для ревью:** Просто читайте файлы в указанном выше порядке. Все ответы содержат обоснование и контекст.
*   **Для проверки спецификации API:** Файл `api-design/hotel-api-oas3.yaml` можно загрузить в Swagger Editor или Postman для визуализации.
*   **Запуск скриптов (по желанию):** Для кейсов требуются Python 3.x и библиотека `requests`. Инструкции внутри.

## Ключевые подходы, применённые в работе

*   **Инженерный подход:** Проблемы анализируются через призму системы, логирования и воспроизводимости.
*   **Коммуникация с акцентом на клиента:** Ответы партнёрам составлены в деловом и в то же время дружелюбном тоне, с ясными шагами по решению.
*   **Проактивность:** В каждом разделе даны рекомендации по улучшению процессов и предотвращению подобных инцидентов.
