## Моё проектирование API отельного продукта

Минимальный набор методов, необходимых для обеспечения основных бизнес-процессов системы бронирования отелей:

| Функциональность | HTTP-метод | Эндпоинт | Обоснование |
|------------------|------------|----------|-------------|
| **Поиск отелей** | GET | `/api/hotels` | Базовый поиск с фильтрами (город, даты, гости). Без поиска невозможно начать процесс бронирования. |
| **Детали отеля и его доступность** | GET | `/api/hotels/{id}/rooms` | Получение информации о конкретном отеле, типах номеров, ценах и наличии на выбранные даты. Ключевой этап перед бронированием. |
| **Создание сессии бронирования** | POST | `/api/booking-forms` | Резервирование номера на короткое время, получение `book_hash` для продолжения. Защищает от двойного бронирования и фиксирует условия. |
| **Подтверждение бронирования** | POST | `/api/orders` | Финальное создание заказа с данными гостей и оплаты. Это ключевой бизнес-процесс, завершающий сделку. |
| **Получение информации о брони** | GET | `/api/orders/{id}` | Просмотр деталей заказа, статуса, условий отмены. Необходимо для информирования гостя и самообслуживания. |
| **Отмена бронирования** | DELETE | `/api/orders/{id}` | Важная функция для управления заказами. Реализована через DELETE как идемпотентный метод, обеспечивающий предсказуемость. |

**Обоснование данного набора методов и эндпоинтов**

Этот набор методов покрывает полный пользовательский путь (Customer Journey) от поиска отеля до отмены брони. Он включает двухэтапное бронирование (Сессия + Подтверждение), что является отраслевым стандартом для безопасности финансовых транзакций и предотвращения конфликтов.
