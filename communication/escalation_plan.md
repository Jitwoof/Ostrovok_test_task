## Кейс
_"Представь, что активный партнёр недоволен возможностями API и просит доработать его. Опиши свои действия в такой ситуации."_

---

## Мой ответ

Это план действий для ситуации, когда активный Партнёр недоволен возможностями API и просит доработать его:

| Этап | Действия | Цель |
|------|----------|------|
| **1. Сбор информации** | • Выслушать Партнёра, проявить эмпатию.<br>• Запросить конкретные примеры: логи, request_id, сценарии использования.<br>• Уточнить бизнес-задачу, которую не решает текущий API.<br>• Получить полный контекст проблемы, отделить эмоции от фактов. | Понимание полного контекста проблемы, чёткое отделение эмоциональной оценки от фактических данных. |
| **2. Анализ и классификация** | • Воспроизвести проблему в тестовом окружении.<br>• Определить тип: баг, недостаток документации, запрос на новую функциональность.<br>• Оценить сложность и влияние на бизнес. | Чётко классифицировать запрос для определения приоритетов и правильного направления работы. |
| **3. Внутренняя коммуникация** | • Создать тикет с полным описанием для команды разработки.<br>• Обсудить с продакт-менеджером приоритет и сроки.<br>• Если возможен временный обход проблемы, то предложить его команде и партнёру. | Запустить процесс решения, оценить необходимые ресурсы и реалистичные сроки. |
| **4. Коммуникация с Партнёром** | • Дать обратную связь в оговорённые сроки (в т.ч. статус "приняли в работу").<br>• Сообщить о сложностях и сроках, если есть препятствия.<br>• Предложить временный обход проблемы, если он есть.<br>• Договориться о следующем контакте. | Поддерживать доверие, управлять ожиданиями, снижать недовольство за счёт прозрачности. |
| **5. Завершение** | • Проинформировать Партнёра об устранении проблемы.<br>• Проверить, что решение работает у Партнёра.<br>• Обновить документацию при необходимости.<br>• Добавить решение в Базу Знаний для будущих случаев. | Полное решение проблемы и принятие превентивных мер для предотвращения повтора. |

**Ключевой принцип** — Регулярные обновления. Даже если нет прогресса, все равно необходимо информировать Партнёра о статусе.
